コラム 「Rails 3.2/4.0 のどっちから読み始めればいい？」

2013年8月の現時点では、Rails 3.2 と Rails 4.0 のどちらも、Rails コミュニティによって公式にサポートされています。したがって、「以前まで使われていた Rails 3.2 と、最新版である Rails 4.0 のどっちから読めば良いのか？」というのは自然な疑問です。

結論から述べると、「どちらから読めば良いのか？」という疑問の答えは、あなたの今までの開発経験によって異なります。もし今まで Rails を使ったことがないのであれば、Rails 3.2 対応版から読み進めることをお勧めします。というのも、Rails 4.0 はリリースされたばかりで、Rails 4.0 のエコシステムがまだ少し不安定だからです。

また、Rails チュートリアルにおける各バージョンの違いも、ほんの少ししかありません。具体的には、“strong parameters” と呼ばれる Rails 4.0 の新機能の解説があるかどうかが、Rails チュートリアルの各バージョンの主な違いです。

加えて、Rails 4.0 への移行をより簡単にするために、本書には、第12章 (現在翻訳中) とスクリーンキャスト (英語) が用意されています。これらの補足的な資料では、Strong Parameter やセキュリティ上の重要な更新などについて触れています。このため、まずは本書の Rails 3.2 対応版を使って、Web アプリケーションの開発を学ぶことをお勧めします (その際、もし必要であれば、スクリーンキャストも使ってください)。その後、先ほど紹介した補足資料を使って、Rails 3.2 と Rails 4.0 との差分を学習してください。

一方、もしあなたが Rails の開発経験を既に持っていて、かつ、 Rails の最前線の状況を味わってみたいのであれば、 是非 Rails 4.0 対応版 を読んでみてください。 Rails 4.0 対応版では、全編を通して Rails 4.0 が使われています。なお、 Rails 4.0 対応版のコラム 1.1 では、Rails 3.2 と Rails 4.0 との違いをリスト形式でまとめています。 ほぼ全ての違いが網羅的にまとめられているので、こちらも合わせてご活用ください。
